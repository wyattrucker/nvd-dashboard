Vulnerability Status NVD Pipeline

Tracks where a CVE is in NVD's analysis process
Received Just submitted — no analysis started yet
Deferred NVD has deprioritized analysis — often third-party or disputed submissions
Awaiting Analysis Queued for NVD review
Undergoing Analysis NVD is actively scoring it now
Analyzed Fully reviewed, scored, and enriched with CVSS data
Modified Previously analyzed, then updated with new information
Rejected Determined to be a duplicate or invalid entry

KEV CISA Catalog

Known Exploited Vulnerabilities — the highest-priority signal
⚠ Actively Exploited in the Wild

A CVE marked as KEV has been confirmed exploited by real attackers — it is no longer a theoretical risk.

The KEV catalog is maintained by CISA (Cybersecurity and Infrastructure Security Agency). Federal agencies are legally required to patch KEV vulnerabilities by a set deadline. For any organization, KEV status should be treated as the highest remediation priority regardless of CVSS score.

Source: cisa.gov/known-exploited-vulnerabilities-catalog

CVSS Score Severity Rating

Common Vulnerability Scoring System — rates how severe a vulnerability is (0–10)
Range Severity What it means
0.1 – 3.9 Low Limited impact, often requires local access
4.0 – 6.9 Medium Meaningful risk, may need special conditions
7.0 – 8.9 High Significant — remotely exploitable in many cases
9.0 – 10.0 Critical Remote exploit, no auth needed, full impact — patch immediately

CWE Weakness Type

Common Weakness Enumeration — the root cause category of a vulnerability

If a CVE is what happened, the CWE is why it happened. Common examples:

CWE-79 Cross-Site Scripting — malicious scripts injected into webpages
CWE-89 SQL Injection — attacker manipulates database queries
CWE-22 Path Traversal — accessing files outside the intended directory
CWE-287 Improper Authentication — system fails to verify identity
CWE-476 NULL Pointer Dereference — code crashes on unhandled null values
